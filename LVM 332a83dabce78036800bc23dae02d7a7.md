# LVM — База

---

## Начало

**LVM = 3 уровня абстракции:**

```
Физический диск/раздел → Physical Volume (PV)
        ↓
  Пул из PV-шников   → Volume Group (VG)
        ↓
  Логический раздел  → Logical Volume (LV)
```

**Ключевые понятия:**

- **PE (Physical Extent)** — минимальная единица выделения пространства в VG (по умолчанию 4 MB)
- **LE (Logical Extent)** — аналог PE на уровне LV, 1:1 маппинг
- **Thin provisioning** — выделение пространства «по требованию», LV может быть больше реального места в VG
- **Snapshot** — мгновенный слепок LV на момент создания (copy-on-write)

**Минимальный рабочий сценарий** — создать LV с нуля:

```bash
# 1. Инициализируем диск как PV
pvcreate /dev/sdb

# 2. Создаём Volume Group
vgcreate vg_data /dev/sdb

# 3. Создаём Logical Volume (например, 50 GB)
lvcreate -L 50G -n lv_data vg_data

# 4. Форматируем и монтируем
mkfs.ext4 /dev/vg_data/lv_data
mount /dev/vg_data/lv_data /mnt/data

# 5. Прописываем в /etc/fstab для автомонтирования
echo '/dev/vg_data/lv_data /mnt/data ext4 defaults 0 2' >> /etc/fstab
```

```
> **Путь к LV всегда:** `/dev/<VGname>/<LVname>` или симлинк `/dev/mapper/<VGname>-<LVname>`
```

## Топ-10 команд

```bash
# 1. Показать состояние всех PV/VG/LV (краткий вывод)
pvs && vgs && lvs

# 2. Детальная информация о Volume Group
vgdisplay vg_data

# 3. Расширить LV на +20G и сразу ресайзнуть FS (флаг -r)
lvextend -r -L +20G /dev/vg_data/lv_data

# 4. Расширить LV до 100% свободного места в VG
lvextend -r -l +100%FREE /dev/vg_data/lv_data

# 5. Добавить новый диск в существующий VG
pvcreate /dev/sdc && vgextend vg_data /dev/sdc

# 6. Создать снапшот LV (нужно свободное место в VG)
lvcreate -s -L 10G -n lv_data_snap /dev/vg_data/lv_data

# 7. Откатиться к снапшоту (LV должен быть размонтирован)
umount /mnt/data
lvconvert --merge /dev/vg_data/lv_data_snap

# 8. Безопасно удалить LV
umount /mnt/data && lvremove /dev/vg_data/lv_data

# 9. Перенести данные с одного PV на другой (для замены диска)
pvmove /dev/sdb /dev/sdc
vgreduce vg_data /dev/sdb
pvremove /dev/sdb

# 10. Мониторинг thin pool (заполненность данных и метаданных)
lvs -o +data_percent,metadata_percent vg_data/thin_pool
```

## Таблица-шпаргалка

### Ключевые флаги `lvcreate` / `lvextend`

| Флаг | Значение | Пример |
| --- | --- | --- |
| `-L 20G` | Абсолютный размер | `lvcreate -L 20G` |
| `-L +20G` | Добавить к текущему | `lvextend -L +20G` |
| `-l +100%FREE` | Всё свободное место VG | `lvextend -l +100%FREE` |
| `-l 50%VG` | 50% от размера VG | `lvcreate -l 50%VG` |
| `-n` | Имя LV | `-n lv_data` |
| `-r` | Авторесайз FS после изменения | `lvextend -r -L +10G` |
| `-s` | Создать снапшот | `lvcreate -s ...` |
| `-m1` | Зеркало (RAID1) | `lvcreate -m1 -L 10G` |
| `--thinpool` | Thin provisioning pool | `lvcreate --thinpool` |
| `-Z y/n` | Обнуление первых блоков LV | `lvcreate -Z y` |

### Команды по уровням

| Действие | PV | VG | LV |
| --- | --- | --- | --- |
| **Создать** | `pvcreate` | `vgcreate` | `lvcreate` |
| **Показать (кратко)** | `pvs` | `vgs` | `lvs` |
| **Показать (детально)** | `pvdisplay` | `vgdisplay` | `lvdisplay` |
| **Расширить** | — | `vgextend` | `lvextend` |
| **Уменьшить** | — | `vgreduce` | `lvreduce` |
| **Удалить** | `pvremove` | `vgremove` | `lvremove` |
| **Переименовать** | — | `vgrename` | `lvrename` |
| **Сканировать** | `pvscan` | `vgscan` | `lvscan` |
| **Активировать** | — | `vgchange -ay` | `lvchange -ay` |

### Типы LV

| Тип | Команда создания | Когда использовать |
| --- | --- | --- |
| **Linear** | `lvcreate -L 10G` | Стандартный случай |
| **Striped** | `lvcreate -i2 -I64 -L 10G` | Производительность (RAID0) |
| **Mirror** | `lvcreate -m1 -L 10G` | Надёжность (RAID1) |
| **Snapshot** | `lvcreate -s -L 5G` | Бэкап, тестирование |
| **Thin LV** | `lvcreate -T --thinpool` | Оверпровизионинг |
| **Cache LV** | `lvcreate --type cache` | SSD-кэш для HDD |

## Практические сценарии

### Расширение диска (самый частый кейс)

```bash
# Ситуация: /dev/vg_data/lv_data заполнен, нужно добавить место

# Шаг 1: Проверяем свободное место в VG
vgs vg_data

# Шаг 2a: Если место есть в VG — просто расширяем
lvextend -r -L +20G /dev/vg_data/lv_data

# Шаг 2b: Если места нет — добавляем новый диск
pvcreate /dev/sdc
vgextend vg_data /dev/sdc
lvextend -r -l +100%FREE /dev/vg_data/lv_data
```

### Снапшот для безопасного бэкапа

```bash
# Создаём снапшот
lvcreate -s -L 10G -n lv_data_snap /dev/vg_data/lv_data

# Монтируем снапшот (только чтение)
mount -o ro /dev/vg_data/lv_data_snap /mnt/snap

# Делаем бэкап
rsync -av /mnt/snap/ /backup/data/

# Убираем снапшот
umount /mnt/snap
lvremove /dev/vg_data/lv_data_snap
```

### Замена физического диска

```bash
# Переносим данные со старого диска на новый
pvmove /dev/sdb /dev/sdc

# Убираем старый диск из VG
vgreduce vg_data /dev/sdb

# Деинициализируем PV
pvremove /dev/sdb
```

### Уменьшение LV (ext4, только offline!)

```bash
# ВНИМАНИЕ: XFS уменьшить нельзя вообще!

# Шаг 1: Размонтируем
umount /mnt/data

# Шаг 2: Проверяем FS
e2fsck -f /dev/vg_data/lv_data

# Шаг 3: Уменьшаем FS (сначала FS, потом LV!)
resize2fs /dev/vg_data/lv_data 30G

# Шаг 4: Уменьшаем LV
lvreduce -L 30G /dev/vg_data/lv_data

# Шаг 5: Монтируем обратно
mount /dev/vg_data/lv_data /mnt/data
```

---

## ⚠️ Критические правила

- **Уменьшение LV — только offline** и только для ext4; **XFS уменьшить нельзя**
- Порядок при уменьшении: **сначала FS → потом LV**. Обратный порядок = потеря данных
- **Всегда оставляй 10–20% свободного места** в VG для снапшотов и экстренных операций
- Снапшот становится **невалидным**, если переполняется — следи за `lvs -o +data_percent`
- При использовании `pvmove` — **не прерывай процесс**, данные будут в неконсистентном состоянии
- Флаг `r` у `lvextend` работает только с **смонтированными** FS; без него — ресайзить вручную через `resize2fs` или `xfs_growfs`
- **`xfs_growfs`** — расширение XFS; **`resize2fs`** — расширение ext4
