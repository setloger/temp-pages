# Сниффер

---

## Что такое сниффер

Сниффер (network sniffer) — программа или устройство, которое перехватывает пакеты данных, проходящие через сетевой интерфейс. Он становится «посредником» между приложением и сервером, собирает и анализирует HTTP/HTTPS-запросы, заголовки, статус-коды и тела ответов.[1](about:blank#fn1)[2](about:blank#fn2)

В DevOps снифферы используются для:

- Диагностики сетевых проблем и узких мест (troubleshooting)[3](about:blank#fn3)
- Анализа взаимодействия микросервисов
- Проверки корректности передаваемых данных[4](about:blank#fn4)
- Тестирования межсетевых экранов и правил firewall[5](about:blank#fn5)
- Записи дампов трафика для последующего анализа[6](about:blank#fn6)

---

## Основные инструменты

| Инструмент | Тип | Когда использовать |
| --- | --- | --- |
| **tcpdump** | CLI | На сервере без GUI, быстрый захват [7](about:blank#fn7) |
| **Wireshark** | GUI | Детальный визуальный анализ .pcap файлов [8](about:blank#fn8) |
| **sngrep** | CLI | Анализ SIP-трафика (VoIP) [9](about:blank#fn9) |
| **Fiddler / Charles** | GUI Proxy | HTTP/HTTPS на уровне приложения [10](about:blank#fn10) |

## tcpdump — главный инструмент

Это консольная утилита для Unix-систем, которую DevOps использует прямо на сервере.[11](about:blank#fn11)

**Установка:**

```bash
apt-get install tcpdump   # Debian/Ubuntu
yum install tcpdump       # RHEL/CentOS
```

**Базовые команды:**

```bash
# Посмотреть доступные интерфейсы
tcpdump -D

# Слушать все интерфейсы
tcpdump -i any

# Слушать конкретный интерфейс
tcpdump -i eth0

# Фильтр по хосту
tcpdump -i eth0 host 192.168.0.100

# Фильтр по порту и протоколу
tcpdump -i eth0 tcp port 443

# Только ICMP (пинги)
tcpdump -i eth0 icmp
```

**Сохранение и анализ дампов:**

```bash
# Записать трафик в файл .pcap
tcpdump -i eth0 -w /tmp/capture.pcap

# Прочитать сохранённый дамп
tcpdump -nr capture.pcap

# Открыть дамп в Wireshark
wireshark /tmp/capture.pcap
```

## Основные флаги tcpdump

| Флаг | Что делает |
| --- | --- |
| `-i eth0` | Указать интерфейс |
| `-nn` | Не резолвить IP и порты (быстрее) |
| `-A` | Показывать тело пакета в ASCII |
| `-s 0` | Захватывать пакет целиком |
| `-c 500` | Остановиться после N пакетов |
| `-w /tmp/cap.pcap` | Сохранить дамп в файл |
| `-r cap.pcap` | Читать дамп из файла |
| `not port 22` | Исключить SSH из захвата |

---

## TCP-флаги в выводе

| Флаг | Значение |
| --- | --- |
| `[S]` | SYN — клиент открывает соединение |
| `[S.]` | SYN-ACK — сервер ответил ✅ |
| `[.]` | ACK — рукопожатие завершено |
| `[R]` | RST — сброс соединения ⚠️ |
| `[F]` | FIN — нормальное закрытие |

## Wireshark фильтры

```
tcp.flags.reset == 1           # RST пакеты — плохой знак
tcp.flags.syn == 1             # Только SYN (новые соединения)
tcp.analysis.retransmission    # Ретрансмиссии (потери пакетов)
ip.addr == 10.0.0.5            # Трафик конкретного хоста
tcp.port == 5432               # PostgreSQL
http.request                   # HTTP запросы
```

## Wireshark — визуальный анализ

Wireshark открывает `.pcap`-файлы, захваченные `tcpdump` на сервере. Ключевые фильтры отображения (display filters):[12](about:blank#fn12)

```
http.request          — только HTTP-запросы
tls.handshake         — TLS-рукопожатия
tcp.port == 8080      — трафик на порт 8080
ip.addr == 10.0.0.1   — трафик от/до IP
```

> ⚡ **Важно:** capture filters (например, `port 443`) задаются при захвате в `tcpdump`, а display filters применяются уже при анализе дампа в Wireshark.[13](about:blank#fn13)
>

## Практический сценарий

**Задача:** сервис A не может достучаться до сервиса B на порту 5432 (PostgreSQL).

1. Запускаем захват на сервере B:

```bash
sudo tcpdump -i eth0 tcp port 5432 -w /tmp/pg_debug.pcap
```

1. Воспроизводим проблему — делаем запрос из сервиса A
2. Останавливаем захват (`Ctrl+C`), скачиваем дамп на локальную машину:

```bash
scp user@server-b:/tmp/pg_debug.pcap ./
```

1. Открываем в Wireshark и смотрим: есть ли TCP SYN от сервиса A? Есть ли ответный SYN-ACK? Если SYN есть, а SYN-ACK нет — проблема в firewall или сервис не слушает порт[14](about:blank#fn14)[15](about:blank#fn15)
