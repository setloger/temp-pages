# Linux: **Cron & Systemd Timers**

---

## 1. Синтаксис Cron

Каждая строка crontab состоит из пяти полей времени и команды:

```
* * * * *  /path/to/command
│ │ │ │ │
│ │ │ │ └── день недели  (0–7, 0 и 7 = воскресенье)
│ │ │ └──── месяц        (1–12)
│ │ └────── день месяца  (1–31)
│ └──────── час          (0–23)
└────────── минута        (0–59)
```

### Примеры

```bash
# Каждый день в 02:30
30 2 * * *  /usr/bin/backup.sh

# Каждые 15 минут
*/15 * * * *  /usr/bin/check.sh

# По понедельникам в 08:00
0 8 * * 1  /usr/bin/report.sh

# 1-го числа каждого месяца в полночь
0 0 1 * *  /usr/bin/monthly.sh

# В 9:00 и 18:00 по будням (пн–пт)
0 9,18 * * 1-5  /usr/bin/notify.sh
```

## 2. Анатомия Systemd Timer

Systemd требует два файла: `.service` (что запускать) и `.timer` (когда запускать).
Оба размещаются в `/etc/systemd/system/`.

### `/etc/systemd/system/myjob.service`

```
[Unit]
Description=My Scheduled Job

[Service]
Type=oneshot
ExecStart=/usr/bin/backup.sh
User=root
```

### `/etc/systemd/system/myjob.timer`

```
[Unit]
Description=Run myjob every day at 02:30

[Timer]
OnCalendar=*-*-* 02:30:00
Persistent=true

[Install]
WantedBy=timers.target
```

> `Persistent=true` — если система была выключена в момент запуска, задача выполнится сразу после следующей загрузки.
> 

### Активация таймера

```bash
systemctl daemon-reload
systemctl enable --now myjob.timer
```

## 3. Сравнение: Cron vs Systemd Timers

| Критерий | Cron | Systemd Timer |
| --- | --- | --- |
| **Точность** | До минуты | До секунды (`OnCalendar=*:0/30`) |
| **Пропущенный запуск** | Пропускается навсегда | Выполняется при загрузке (`Persistent=true`) |
| **Логирование** | Нет встроенного (`>> logfile 2>&1`) | Полная интеграция с `journald` |
| **Зависимости** | Отсутствуют | `After=network.target`, `Requires=db.service` |
| **Управление ресурсами** | Нет | `CPUQuota=`, `MemoryMax=` через cgroups |
| **Сложность настройки** | Одна строка в crontab | Два файла + `daemon-reload` |
| **Переносимость** | Универсален (любой Linux/Unix) | Только systemd-системы |

## 4. Команды управления

### Cron

```bash
# Редактировать задачи текущего пользователя
crontab -e

# Просмотреть список задач
crontab -l

# Системные задачи
cat /etc/crontab
ls /etc/cron.d/

# Логи (зависит от дистрибутива)
grep CRON /var/log/syslog
journalctl -t cron
```

### Systemd Timers

```bash
# Список всех активных таймеров
systemctl list-timers --all

# Статус конкретного таймера
systemctl status myjob.timer

# Логи задачи через journald
journalctl -u myjob.service
journalctl -u myjob.service --since "1 hour ago"

# Перезапуск таймера
systemctl restart myjob.timer

# Ручной запуск сервиса (вне расписания)
systemctl start myjob.service

# Отключить таймер
systemctl disable --now myjob.timer
```

## 5. Когда что использовать

### Используй Cron, если:

- Нужна простая задача в одну строку без зависимостей
- Работаешь со старыми системами или скриптами, которые должны быть портабельны
- Задачи управляются на уровне пользователя (`crontab -e`)
- Команда незнакома с systemd

### Переходи на Systemd Timers, если:

- Нужна гарантия выполнения пропущенных задач (бэкапы, отчёты)
- Задача зависит от сети, базы данных или другого сервиса
- Критично логирование и отладка (`journalctl -u`)
- Нужно ограничить потребление CPU/памяти для задачи
- Расписание сложнее стандартного cron (например, «каждые 30 секунд» или «первый понедельник месяца»)
