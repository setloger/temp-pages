## Что такое `/proc`?

>`/proc` — это **виртуальная псевдофайловая система** (procfs), которая живёт только в **оперативной памяти**.
> Никакие файлы **не хранятся на диске** — каждый файл генерируется ядром **«на лету»** в момент чтения.

```bash
mount | grep proc
# proc on /proc type proc (rw,nosuid,nodev,noexec,relatime)
```

**Кто использует `/proc`:**

`ps` · `top` · `htop` · `free` · `netstat` · `lsof` — все берут данные отсюда.

---

## Структура `/proc`

```
/proc/
├── <PID>/          # Директория каждого запущенного процесса
│   ├── cmdline     # Командная строка запуска
│   ├── status      # Состояние, UID, GID, память
│   ├── fd/         # Открытые файловые дескрипторы (симлинки)
│   ├── maps        # Карта памяти процесса (mmap-регионы)
│   ├── environ     # Переменные окружения
│   ├── net/        # Сетевые сокеты процесса
│   └── cgroup      # К каким cgroup принадлежит
│
├── cpuinfo         # Информация о CPU, ядрах, флагах
├── meminfo         # RAM, swap, кеши, буферы
├── mounts          # Текущие точки монтирования
├── partitions      # Блочные устройства и разделы
├── filesystems     # Поддерживаемые типы ФС
├── loadavg         # Средняя нагрузка (1/5/15 мин)
├── uptime          # Время работы системы (секунды)
├── version         # Версия ядра
├── net/            # TCP, UDP, ARP, маршруты
└── sys/            # ⚠️ Управление ядром на лету (read/write)
```

## `/proc/sys/` — Изменение параметров ядра без перезагрузки

| Раздел | Что настраивает |
| --- | --- |
| `/proc/sys/net/` | Сеть: IP-forwarding, TCP-параметры |
| `/proc/sys/vm/` | Виртуальная память, swap, кеши |
| `/proc/sys/kernel/` | Параметры ядра: hostname, лимиты |
| `/proc/sys/fs/` | Лимиты ФС: file-max, inotify |

```bash
# Включить IP-forwarding (Docker, VPN, роутинг)
echo 1 > /proc/sys/net/ipv4/ip_forward

# Защита от SYN-flood
echo 2048 > /proc/sys/net/ipv4/tcp_max_syn_backlog

# Настроить swappiness
echo 10 > /proc/sys/vm/swappiness

# Сбросить page cache
echo 3 > /proc/sys/vm/drop_caches
```

> Изменения через `/proc/sys/` **не переживают перезагрузку**!
> 
> 
> Для постоянного эффекта → `/etc/sysctl.conf` + `sysctl -p`
>

## Итого

| Возможность | Описание |
| --- | --- |
| **Мониторинг** | Читать состояние любого процесса и системы |
| **Отладка** | FD, maps, environ — без ptrace и strace |
| **Управление** | Менять параметры ядра через `/proc/sys/` |
| **Изоляция** | Namespaces дают контейнерам свой `/proc` |
| **Инструменты** | ps, top, htop, free, netstat читают именно отсюда |
